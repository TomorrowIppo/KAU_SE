# 요구사항 정의서


## 1. 서론
### 1.1 문서의 목적 및 범위
본 문서는 재난 상황에서 기지국 파괴 시 드론 간 P2P 메쉬 네트워크를 구축하여 긴급 통신망을 제공하는 시스템의 요구사항을 정의한다.

### 1.2 대상 시스템 개요
중앙 서버 없는 탈중앙화 구조로 드론 노드들이 스스로 경로를 탐색하여 데이터(텍스트, GPS, 이미지)를 전송하는 시스템.

## 2. 요구사항
### 2.1 기능적 요구사항 (FR)
- FR-001: 드론 간 동적 경로 탐색 및 릴레이 전송 구현.
- FR-002: 노드 이탈 시 5초 이내 네트워크 경로 자동 재구성.
- FR-003: 신호 복구 시 별도 재부팅 없이 망 자동 재합류.
- FR-004: 텍스트, GPS, 저대역폭 이미지 실시간 전송.

### 2.2 비기능적 요구사항 (NFR)
- NFR-001: End-to-End 지연 시간 최대 500ms 이내 (성능).
- NFR-002: 배터리 10% 미만 시 구조 메시지 외 데이터 차단 (신뢰성).
- NFR-003: Linux/ROS 환경 호환성 유지 (이식성).
- NFR-004: 승인된 단말기만 접속 허용 (보안성).
