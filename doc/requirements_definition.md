# 요구사항 정의서 (Requirement Definition) v1.1

## 1. 문서 개요

### 1.1 목적
- 본 문서는 '클라우드 파일 공유 시스템(미니 드라이브)'의 요구사항을 정의하여 시스템 개발의 범위를 결정하고 검증의 기준을 제공한다.

### 1.2 범위
- 조직 내 200명 사용자를 위한 파일 업로드, 다운로드, 폴더 관리, 공유, 버전 관리 및 검색 기능을 포함한다.

### 1.3 관련 문서
- project_management_plan.md
- project_definition.md
- quality_definition.md

--------------------------------------------------

## 2. 요구사항 작성 원칙 

- **요구사항 정의 품질 요소**: 정확성, 모호성 없음, 완전성, 일관성, 추적성
- **요구사항 작성 지침**: 
    - 모든 요구사항에 유일한 식별자(ID) 부여 (FR: 기능, NFR: 비기능, IR: 인터페이스, DR: 개발)
    - 문장의 5형식에 따른 단문 작성 및 우선순위(상/중/하) 부여

--------------------------------------------------

## 3. 기능적 요구사항 (Functional Requirements)

| ID | 분류 | 요구사항 상세 내용 | 우선순위 |
| :--- | :--- | :--- | :---: |
| FR-001 | 파일 관리 | 사용자는 다양한 형식의 파일을 시스템에 업로드한다. | 상 |
| FR-002 | 파일 관리 | 사용자는 업로드된 파일을 자신의 기기로 다운로드한다. | 상 |
| FR-003 | 폴더 관리 | 사용자는 다단계 계층 구조의 폴더를 생성한다. | 상 |
| FR-004 | 폴더 관리 | 사용자는 파일을 다른 폴더로 이동하거나 삭제한다. | 상 |
| FR-005 | 폴더 관리 | 사용자는 삭제된 파일을 휴지통에서 30일 동안 보관 및 확인한다. | 중 |
| FR-006 | 공유/협업 | 사용자는 특정 사용자에게 '읽기 전용' 권한을 부여한다. | 상 |
| FR-007 | 공유/협업 | 사용자는 특정 사용자에게 '수정 가능' 권한을 부여한다. | 상 |
| FR-008 | 공유/협업 | 사용자는 만료 기간이 포함된 파일 공유 링크를 생성한다. | 상 |
| FR-009 | 버전 관리 | 시스템은 파일 수정 시 이전 버전을 저장하고 사용자의 요청 시 특정 시점으로 복구한다. | 중 |
| FR-010 | 검색 기능 | 사용자는 파일 이름을 기준으로 원하는 파일을 검색한다. | 상 |
| FR-011 | 검색 기능 | 사용자는 파일 유형 및 날짜별로 검색 결과를 필터링한다. | 중 |
| FR-012 | 계정 관리 | 사용자는 이메일 정보를 입력하여 시스템 회원가입을 수행한다. | 상 |
| FR-013 | 계정 관리 | 사용자는 등록된 이메일과 비밀번호로 시스템에 로그인한다. | 상 |
| FR-014 | 계정 관리 | 사용자는 이용 종료 후 시스템에서 안전하게 로그아웃한다. | 상 |
| FR-015 | 제약 사항 | 시스템은 단일 파일의 업로드 크기를 최대 1GB로 제한한다. | 중 |
| FR-016 | 제약 사항 | 시스템은 동시 파일 업로드 개수를 최대 10개로 제한한다. | 하 |
| FR-017 | 예외 처리 | 시스템은 기능 오류 발생 시 사용자에게 명확한 에러 메시지를 출력한다. | 상 |

## 4. 비기능적 요구사항 (Non-Functional Requirements)

| ID | 분류 | 요구사항 상세 내용 | 우선순위 |
| :--- | :--- | :--- | :---: |
| NFR-001 | 성능 | 시스템은 검색 결과를 2초 이내에 사용자에게 출력한다. | 상 |
| NFR-002 | 보안 | 시스템은 모든 전송 데이터를 SSL/TLS로 암호화한다. | 상 |
| NFR-003 | 신뢰성 | 서버 장애 발생 시 24시간 이내에 서비스를 복구한다. | 중 |
| NFR-004 | 사용성 | 시스템은 별도의 프로그램 설치 없이 웹 브라우저에서 동작한다. | 상 |
| NFR-005 | 보안 | 시스템은 사용자 비밀번호를 일방향 암호화(Hash)하여 저장한다. | 상 |

## 5. 인터페이스 및 개발 요구사항 (IR & DR)

### 5.1 인터페이스 요구사항 (IR)
| ID | 분류 | 요구사항 상세 내용 | 우선순위 |
| :--- | :--- | :--- | :---: |
| IR-001 | 사용자 UI | 사용자는 반응형 웹 표준을 준수하는 GUI를 통해 시스템에 접속한다. | 상 |
| IR-002 | 외부 연동 | 시스템은 클라우드 스토리지 API와 데이터를 실시간으로 연동한다. | 상 |

### 5.2 개발 요구사항 (DR)
| ID | 분류 | 요구사항 상세 내용 | 우선순위 |
| :--- | :--- | :--- | :---: |
| DR-001 | SW 스택 | 개발팀은 IntelliJ IDEA, Spring Boot, React, MySQL, GitHub을 사용하여 개발한다. | 중 |
| DR-002 | HW 환경 | 시스템은 RAM 16GB 이상의 PC 및 리눅스 기반 테스트 서버 환경에서 구축한다. | 중 |

## 6. 요구사항 추적 및 변경 관리
- 모든 요구사항은 추적성 매트릭스(RTM)를 통해 분석/설계/테스트 단계와 매핑 관리한다.
- 요구사항 변경 시 영향도 분석 후 베이스라인을 업데이트하며 이력을 기록한다.
