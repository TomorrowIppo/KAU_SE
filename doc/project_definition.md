# Project Definition: 클라우드 파일 공유 시스템 (Mini Drive)

## 1. 프로젝트 개요
본 프로젝트는 조직 내 분산된 파일 관리 문제를 해결하고, 중앙 집중식의 안전한 협업 환경을 구축하기 위한 '클라우드 기반 파일 공유 및 관리 시스템' 개발을 목표로 한다.

## 2. 배경 및 필요성
- **버전 관리의 어려움**: 이메일 및 메신저를 통한 파편화된 공유로 인해 최신본 식별 불가 및 작업 혼선 발생.
- **검색 효율성 저하**: 파일이 여러 경로에 분산되어 파일 탐색에 과도한 시간 소요.
- **보안 취약성**: 외부 업체와의 공유 시 권한 제어 및 유출 방지 대책 부재.

## 3. 주요 목표 (Overall Goals)
- 조직 내 모든 파일의 중앙 집중식 관리 체계 구축.
- 직관적인 UI/UX를 통한 쉬운 업로드 및 다운로드 환경 제공.
- 사용자 간 권한 기반 파일 공유 및 협업 지원.
- 빠르고 정확한 파일 검색 기능 제공.

## 4. 핵심 기능 (Core Functionalities)
- **파일/폴더 관리**: 계층형 구조, 이동, 삭제 및 복구.
- **공유 및 권한**: 사용자별 권한(조회/수정/댓글) 설정 및 만료 기간이 포함된 외부 공유 링크 생성.
- **검색 시스템**: 파일명, 날짜, 소유자, 유형별 상세 필터링.
- **버전 관리**: 수정 이력 추적, 이전 버전 다운로드 및 복원.
- **사용자 관리**: 이메일 기반 인증 및 관리자 전용 저장 공간 제어 기능.

## 5. 제약 사항 및 기술 요구사항
- **대상**: 약 200명의 동시 사용자 수용.
- **접근성**: 별도 설치가 필요 없는 웹 브라우저 기반 서비스 지향.
- **보안**: 전송 시 암호화 및 권한 기반 접근 제어(RBAC) 적용.
