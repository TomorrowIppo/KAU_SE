# 프로젝트 정의서: 미니 드라이브 (Mini-Drive)

## 1. 프로젝트 개요
본 프로젝트는 조직 내 분산된 파일을 중앙에서 관리하고, 팀원 간의 효율적인 협업 및 안전한 파일 공유를 지원하기 위한 클라우드 기반 파일 관리 시스템 구축을 목표로 한다.

## 2. 추진 배경 및 필요성
* **버전 관리의 어려움**: 동일 파일이 여러 경로에 산재하여 최신본 확인 및 작업 혼선 발생.
* **검색 효율성 저하**: 파일 수 증가에 따른 파편화로 인해 데이터 탐색 시간 낭비.
* **보안 취약성**: 외부 업체와 파일 공유 시 접근 권한 제어 불능 및 유출 위험 존재.

## 3. 주요 기능 범위 (Scope)
* **파일 관리**: 업로드/다운로드, 폴더 생성 및 이동, 삭제 파일 복구.
* **버전 관리**: 수정 이력 보관, 이전 버전 다운로드 및 복원 기능.
* **공유 및 권한**: 사용자별 권한(읽기/편집/댓글) 설정, 외부 공유용 만료 링크 생성.
* **검색 시스템**: 파일명, 업로드 날짜, 업로드 사용자, 파일 유형별 상세 검색.
* **계정 관리**: 이메일 기반 로그인 및 보안 인증.

## 4. 기대 효과
* 중앙 집중식 관리로 데이터 무결성 및 최신성 확보.
* 강력한 검색 기능을 통한 업무 생산성 향상.
* 권한 제어 및 만료 링크를 통한 조직 정보 보안 강화.
