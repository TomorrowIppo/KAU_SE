# 프로젝트 관리 계획서 (Project Management Plan)

## 1. 문서 정보 및 개요
- **문서 목적**: '클라우드 파일 공유 시스템(미니 드라이브)' 개발 프로젝트의 성공적 완수를 위한 관리 전략, 일정, 조직 및 기술적 방안을 정의함.
- **프로젝트 개요**: 조직 내 분산된 파일의 중앙 관리 및 안전한 협업 환경 구축 (동시 사용자 200명 규모).
- **관련 문서**:
    - `project_definition.md` (프로젝트 정의 및 기능 범위)
    - `quality_definition.md` (품질 특성 및 지표 정의)

## 2. 개발 계획
- **프로세스 모델 및 개발 방법론**: 점진적/반복적 모델 (Incremental & Iterative Model). 핵심 기능을 우선 개발 후 부가 기능을 반복적으로 추가함.
- **작업 분할도 (WBS)**:
    1. 환경 설정 및 분석: 프로젝트 착수, 요구사항 명세
    2. 시스템 설계: 아키텍처, DB, UI/UX 설계
    3. 개발 1단계: 사용자 인증 및 기본 파일 CRUD
    4. 개발 2단계: 공유 권한 및 링크 관리 기능
    5. 개발 3단계: 버전 관리 및 상세 검색 기능
    6. 통합 및 테스트: 부하 테스트(200명), 최종 검수
- **소요 자원**: 개발 서버(AWS/GCP), BE/FE 개발자, QA 인력, UI/UX 디자이너.

## 3. 조직 구성
- **팀 구조**: **하이브리드 구조 (Hybrid Structure)**. 기능별 전문성을 유지하면서도 프로젝트 단위의 유연한 대응이 가능하도록 구성.
- **책임 및 역할**:
    - **PM**: 전체 프로젝트 리딩, 리스크 및 일정 총괄.
    - **BE Dev**: API 서버 구축, DB 설계, 클라우드 스토리지 연동.
    - **FE Dev**: 웹 인터페이스 구현, 반응형 레이아웃 적용.
    - **QA**: 품질 목표 달성 검증, 200명 동시성 테스트 수행.

## 4. 기술 관리 계획
- **버전 관리**: **GitHub** 사용 (Git-flow 전략 적용).
- **형상 관리**: 코드뿐만 아니라 설계서, 요구사항 정의서 등 모든 산출물의 변경 이력을 GitHub Repository 및 버전 태깅으로 관리.
- **기술 관리**: 주간 코드 리뷰 수행, 기술 스택(Spring Boot, Kotlin, React) 표준 준수.

## 5. 품질 관리 계획
- **품질 목표**: ISO/IEC 25010 기반 (기능성, 신뢰성, 성능 효율성 최우선).
- **검토 방법 및 주기**: 격주 단위 정기 품질 검토회(Quality Gate) 실시.
- **품질 관리 적용 기법**: 단위 테스트 자동화, 정적 코드 분석, 성능 부하 테스트(JMeter 등).
- **리스크 관리**: 식별된 리스크(일정 지연, 보안 취약점)에 대한 완화 방안 마련 및 월간 업데이트.

## 6. 개발 환경
- **SW 스펙**: IntelliJ IDEA, JDK 17+, Spring Boot 3.x, MySQL 8.0, React, Git/Github.
- **HW 스펙**: 개발 PC (RAM 16GB 이상), 테스트 서버 (vCPU 4, RAM 8GB 이상).
- **개발 공간 및 보안**: 대학 전산실 및 개인 개발 환경 공유, VPN 및 2단계 인증(2FA) 적용.

## 7. 개발 산출물
- 요구사항 명세서, 시스템 설계서(UML 포함), 테스트 결과 보고서, 사용자 및 관리자 매뉴얼, 최종 소스 코드.

## 8. 기타 고려 사항
- AI 기반 파일 자동 분류 기능은 추후 확장 가능하도록 모듈화 설계 반영.
