# 품질 정의서 (Software Quality Specification)

본 문서는 재난 대응이라는 특수한 상황을 고려하여 **신뢰성**, **성능 효율성**, **이식성**을 핵심 지표로 정의한다.

## 1. 핵심 품질 목표
생명 구조와 직결되는 시스템이므로 **무정지 동작(Reliability)**과 **신속한 망 구성(Efficiency)**에 최우선 순위를 둔다.

## 2. 품질 요소별 세부 정의

### 2.1 신뢰성 (Reliability) - [최우선]
* **가용성**: 특정 드론(Node)이 이탈하더라도 전체 네트워크가 5초 이내에 경로를 재구성하여 통신을 유지해야 한다.
* **결함 허용성(Fault Tolerance)**: 하드웨어 결함이나 통신 방해 상황에서도 핵심 긴급 메시지는 반드시 전달되어야 한다.
* **복구 가능성**: 일시적인 전파 간섭 후 신호가 잡히면 별도의 재부팅 없이 자동으로 망에 재합류해야 한다.
* **데이터 우선순위(Priority Queue)**: 배터리 잔량이 10% 미만인 노드 발생 시, 해당 노드는 일반 데이터를 차단하고 '최우선 구조 메시지' 전송 및 노드 교체 신호 발신에만 자원을 집중한다.

### 2.2 성능 효율성 (Performance Efficiency)
* **시간 효율성**: 노드 간 홉(Hop) 수가 증가하더라도 긴급 메시지의 End-to-End 지연 시간은 500ms 이내여야 한다.
* **자원 활용성**: 드론의 제한된 배터리 자원을 고려하여, 통신 프로토콜은 전력 소모를 최소화하는 경량 패킷 구조를 가진다.
* **연결 안정성(Connection Stability)**:  드론 노드 간 이동 중에도 신호 강도(RSSI)를 실시간 모니터링하며, 최대 연결 거리의 80% 도달 시 선제적으로 인접 노드 탐색을 시작하여 끊김 없는 통신을 보장한다.

### 2.3 보안성 (Security)
* **무결성**: 재난 상황을 악용한 가짜 구조 신호나 데이터 변조가 발생하지 않도록 인증 메커니즘을 적용한다.
* **접근 제어**: 승인된 구조 요원의 단말기만 드론 메쉬망에 접속할 수 있도록 관리한다.

### 2.4 이식성 (Portability)
* **적응성**: 다양한 제조사의 드론 하드웨어 및 OS 환경(Linux/ROS 등)에서도 동일한 통신 프로토콜이 동작해야 한다.

## 3. 품질 검증 방법
* **시뮬레이션 테스트**: 10대 이상의 가상 드론 노드 중 일부를 임의로 제거했을 때 망 유지 여부 확인.
* **정량적 지표**: 노드 추가 시 네트워크 구성 완료 시간 및 패킷 손실률(Packet Loss Rate) 측정.
* **현장 적합성 검토**: 이동 속도가 빠른 드론 환경에서의 도플러 효과 및 신호 감쇄 대응력 검증.
