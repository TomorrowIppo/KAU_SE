--------------------------------------------------
1. 서론
--------------------------------------------------
1.1 목적 및 범위
본 문서는 사용자가 개인용 파일을 안전하게 업로드하고 관리할 수 있는 '미니드라이브' 시스템의 요구사항을 정의한다. 본 시스템은 객체지향 설계의 계층화(Layering) 원칙을 적용하여 UI와 로직, 데이터 관리 계층을 분리함으로써 유지보수성을 극대화하는 것을 범위로 한다.

1.2 용어 정의
- DriveItem: 파일과 폴더의 공통적 특징을 추출한 상위 추상 클래스.
- Metadata: 파일의 이름, 크기, 수정일 등 파일 데이터를 설명하는 정보.
- API: 응용 프로그램에서 기능을 제어할 수 있도록 제공하는 인터페이스.

1.3 참조 문서
- ch11_객체지향 설계 강의 자료 
- [샘플] 요구사항 분석서 (open CCTV)

2. 시스템 개요
--------------------------------------------------
2.1 소프트웨어 문맥도
- 본 시스템은 사용자(User)의 요청을 처리하며, 내부적으로 파일 메타데이터를 관리하는 DB와 실제 물리적 파일을 저장하는 클라우드 스토리지로 구성된다.

2.2 기능 분류 및 설명
2.2.1 UseCase Diagram
- Actor Table:
  - 사용자(Primary): 시스템을 통해 파일을 관리하는 주체.
  - 시스템: 파일 업로드/다운로드 및 공유 로직을 수행하는 주체.
  - 외부 스토리지: 실제 파일 데이터가 물리적으로 저장되는 외부 서버.
- UseCase 목록:
  1. 회원가입을 한다. (U_01)
  2. 로그인을 한다. (U_02)
  3. 파일을 업로드한다. (U_03)
  4. 파일을 다운로드한다. (U_04)
  5. 폴더를 생성한다. (U_05)
  6. 공유 링크를 생성한다. (U_06)
  7. 미디어 파일을 재생한다. (U_07)

2.2.2 UseCase Descriptions

[U_01] Use Case Name: 회원가입을 한다.
- ID: U_01 | Importance Level: High
- Primary Actor: 사용자
- Use Case Type: Detail, Essential
- Brief Description: 사용자가 서비스를 이용하기 위해 계정을 생성한다.
- Trigger: 사용자는 회원가입 버튼을 누른다.
- Normal Flow:
  1. 사용자는 아이디, 비밀번호, 이메일을 입력한다.
  2. 사용자는 가입하기 버튼을 누른다.
  3. 시스템은 중복 여부를 확인 후 회원 정보를 저장한다.
- Alternate Flows:
  2.a1: 공란이 있을 경우 시스템은 실패 이유를 출력한다.

[U_02] Use Case Name: 로그인을 한다.
- ID: U_02 | Importance Level: High
- Primary Actor: 사용자
- Brief Description: 계정 정보를 확인하여 시스템 접속 권한을 얻는다.
- Normal Flow:
  1. 사용자는 아이디와 비밀번호를 입력한다.
  2. 시스템은 DB의 정보와 대조하여 일치 여부를 확인한다.
  3. 인증 성공 시 메인 화면으로 이동한다.

[U_03] Use Case Name: 파일을 업로드한다.
- ID: U_03 | Importance Level: High
- Primary Actor: 사용자
- Brief Description: 로컬 기기의 파일을 클라우드 저장소로 전송한다.
- Normal Flow:
  1. 사용자가 업로드할 파일을 선택한다.
  2. 시스템은 가용 용량을 확인 후 파일을 전송한다.
  3. 저장 완료 후 파일 목록을 갱신한다.

[U_04] Use Case Name: 파일을 다운로드한다.
- ID: U_04 | Importance Level: High
- Primary Actor: 사용자
- Brief Description: 클라우드에 저장된 파일을 로컬 기기로 저장한다.
- Normal Flow:
  1. 사용자가 특정 파일을 선택하고 다운로드 버튼을 누른다.
  2. 시스템은 파일 권한을 확인하고 전송을 시작한다.

[U_05] Use Case Name: 폴더를 생성한다.
- ID: U_05 | Importance Level: Medium
- Primary Actor: 사용자
- Brief Description: 파일 관리를 위한 새로운 디렉토리를 생성한다.
- Normal Flow:
  1. 사용자가 새 폴더 이름을 입력한다.
  2. 시스템은 현재 경로에 폴더 객체를 생성한다.

[U_06] Use Case Name: 공유 링크를 생성한다.
- ID: U_06 | Importance Level: Medium
- Primary Actor: 사용자
- Brief Description: 타인과 파일을 공유할 수 있는 고유 URL을 만든다.
- Normal Flow:
  1. 사용자가 공유할 파일을 선택한다.
  2. 시스템은 고유 식별값이 포함된 링크를 생성하여 제공한다.

[U_07] Use Case Name: 미디어 파일을 재생한다.
- ID: U_07 | Importance Level: Low
- Primary Actor: 사용자
- Brief Description: 동영상 등을 다운로드 없이 실시간 재생(Streaming)한다.
- Normal Flow:
  1. 사용자가 미디어 파일을 선택한다.
  2. 시스템은 스트리밍 인터페이스를 통해 데이터를 전송한다.

3. 요구사항 명세
--------------------------------------------------
3.1 정적 분석 (Class Diagram)
- DriveItem (Abstract): name, size, ownerId 속성 포함.
- File (Inheritance): extension 속성 및 download() 메서드 포함.
- Directory (Inheritance): childItems 리스트를 통해 파일 및 폴더 관리.

3.2 CRC 카드 
- Class: FileManager | Responsibility: 파일 CRUD 로직 제어 | Collaborators: DriveItem, StorageManager.
- Class: AuthManager | Responsibility: 사용자 인증 및 권한 확인 | Collaborators: User.

3.3 동적 분석 (Sequence Diagram) 
- 업로드 시나리오: 사용자 -> UI -> FileManager -> StorageManager 순으로 메시지 전송.

--------------------------------------------------
4. 인터페이스 분석
--------------------------------------------------
- HCI 계층: 웹 대시보드 및 파일 탐색기 UI.
- 외부 인터페이스: 클라우드 스토리지 API 연동.
